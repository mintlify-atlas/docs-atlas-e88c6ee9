Get Anchor running locally in minutes using Docker, then create your first note from the web interface.

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) and Docker Compose installed on your machine

## Deploy with Docker

<Steps>
  <Step title="Create a docker-compose.yml">
    Create a new directory for Anchor and add the following `docker-compose.yml` file:

    ```yaml docker-compose.yml
    services:
      anchor:
        image: ghcr.io/zhfahim/anchor:latest
        container_name: anchor
        restart: unless-stopped
        ports:
          - "3000:3000"
        volumes:
          - anchor_data:/data

    volumes:
      anchor_data:
    ```

    <Note>
      Anchor uses an embedded PostgreSQL database by default — no separate database container is required. All data is persisted in the `anchor_data` named volume.
    </Note>
  </Step>

  <Step title="Start the container">
    From the directory containing your `docker-compose.yml`, run:

    ```bash
    docker compose up -d
    ```

    Docker will pull the latest image and start the container in the background. The first startup may take a few seconds while the database initializes.
  </Step>

  <Step title="Open the app">
    Navigate to [http://localhost:3000](http://localhost:3000) in your browser. You should see the Anchor login page.
  </Step>

  <Step title="Create your account">
    Click **Sign up** and register with your name, email address, and a password.

    <Note>
      The first user to register automatically becomes an administrator. As an admin, you can manage users, configure registration settings, and set up OIDC authentication from the admin panel.
    </Note>
  </Step>

  <Step title="Create your first note">
    Once logged in, you'll land on the main notes dashboard.

    1. Click the **New note** button to open the editor.
    2. Give your note a title.
    3. Add content using the rich text editor — supports bold, italic, headings, lists, and checkboxes.
    4. Your note saves automatically as you type and syncs across devices when you sign in elsewhere.

    You can also organize notes with **tags**, **pin** important notes to the top, and **archive** notes you want to keep but not see in your main feed.
  </Step>

  <Step title="Install the mobile app">
    Anchor has an Android app that syncs with your self-hosted instance.

    1. Visit the [GitHub Releases page](https://github.com/zhfahim/anchor/releases).
    2. Download the **Universal APK** (`anchor-{version}.apk`) — this works on all Android devices.
    3. Install the APK, enter the URL of your Anchor instance (e.g. `http://localhost:3000`), and sign in with your account credentials.

    <Note>
      Architecture-specific APKs (arm64, x86_64, etc.) are also available on the releases page if you want a smaller download tailored to your device's CPU.
    </Note>
  </Step>
</Steps>
