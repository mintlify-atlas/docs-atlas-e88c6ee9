---
title: Create tag
description: Create a new tag for the authenticated user.
---

Creates a new tag. Tag names must be unique per user — attempting to create a tag with a name that already exists for the authenticated user will return an error.

## Request body

<ParamField body="name" type="string" required>
  Tag name. Must be non-empty and unique for the authenticated user.
</ParamField>

<ParamField body="color" type="string">
  Color identifier or hex value (e.g. `"#3B82F6"`) to visually distinguish the tag.
</ParamField>

## Response

Returns the created Tag object.

<ResponseField name="id" type="string" required>
  Unique identifier (UUID) of the created tag.
</ResponseField>

<ResponseField name="name" type="string" required>
  Tag name.
</ResponseField>

<ResponseField name="color" type="string">
  Color identifier. `null` if not provided.
</ResponseField>

<ResponseField name="isDeleted" type="boolean" required>
  Always `false` for newly created tags.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 creation timestamp.
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  ISO 8601 timestamp of the last update.
</ResponseField>

<ResponseField name="userId" type="string" required>
  ID of the user who owns this tag.
</ResponseField>

<RequestExample>

```bash cURL
curl --request POST \
  --url 'https://anchor.example.com/api/tags' \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "research",
    "color": "#8B5CF6"
  }'
```

</RequestExample>

<ResponseExample>

```json 201
{
  "id": "tag-uuid-new",
  "name": "research",
  "color": "#8B5CF6",
  "isDeleted": false,
  "createdAt": "2024-03-15T10:00:00.000Z",
  "updatedAt": "2024-03-15T10:00:00.000Z",
  "userId": "user-uuid-here"
}
```

```json 409
{
  "statusCode": 409,
  "message": "A tag with this name already exists"
}
```

</ResponseExample>
